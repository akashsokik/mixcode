# Development Setup

Get MixCode running for development and contribution.

## Prerequisites

- **Node.js** >= 20
- **Bun** (latest)
- **Git**
- Code editor (VS Code recommended)

## Initial Setup

### 1. Fork & Clone

Fork the repository on GitHub, then clone:

```bash
git clone https://github.com/your-username/mixcode.git
cd mixcode
```

### 2. Install Dependencies

```bash
npm install
```

This installs:
- Root dependencies
- Server workspace dependencies
- TUI dependencies (via Bun)

### 3. Create Environment

```bash
cp .env.example .env

# Edit .env with your API keys
# ANTHROPIC_API_KEY=sk-ant-xxxxx
# OPENAI_API_KEY=sk-xxxxx
```

### 4. Verify Installation

```bash
npm run typecheck
```

Should show no errors.

## Development Workflow

### Running Development Servers

**Option 1: Both together**

```bash
npm start
```

Starts backend and TUI in the foreground. Ctrl+C to stop both.

**Option 2: Separate terminals**

Terminal 1 - Backend:
```bash
npm run server
```

Terminal 2 - TUI:
```bash
npm run tui
```

This allows you to restart one without killing the other.

### Making Changes

**Backend changes:**

1. Edit files in `/server/src`
2. Backend auto-restarts (nodemon)
3. Or manually restart: `npm run server`

**TUI changes:**

1. Edit files in `/tui/src`
2. Bun auto-rebuilds
3. Refresh browser (if running in browser mode)
4. Or restart TUI: `npm run tui`

**Shared types:**

1. Edit `/shared/events.ts`
2. Both backend and TUI automatically sync
3. No rebuild needed

### Testing Changes

```bash
# Type checking
npm run typecheck

# Format check
npm run lint

# Run tests (if implemented)
npm run test
```

## IDE Setup

### VS Code

**Extensions:**
- ES7+ React/Redux/React-Native snippets
- TypeScript Vue Plugin
- Prettier
- ESLint

**Settings:**

Create `.vscode/settings.json`:

```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "typescript.tsdk": "node_modules/typescript/lib"
}
```

**Debug Configuration:**

Create `.vscode/launch.json`:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Backend",
      "program": "${workspaceFolder}/node_modules/.bin/tsx",
      "args": ["server/src/index.ts"],
      "console": "integratedTerminal",
      "internalConsoleOptions": "neverOpen"
    },
    {
      "type": "node",
      "request": "launch",
      "name": "Tests",
      "program": "${workspaceFolder}/node_modules/.bin/vitest",
      "console": "integratedTerminal"
    }
  ]
}
```

## Git Workflow

### Branching

```bash
# Create feature branch
git checkout -b feature/new-feature

# Or bug fix
git checkout -b fix/bug-description

# Or documentation
git checkout -b docs/update-guide
```

### Commits

```bash
# Small, focused commits
git add src/file.ts
git commit -m "type: description

Longer explanation if needed.

Fixes #123"
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `refactor`: Code refactoring
- `test`: Tests
- `chore`: Dependencies, config

### Push & PR

```bash
git push origin feature/new-feature
```

Then create PR on GitHub.

interface Session {
  id: string

## Testing

### Add Tests

```typescript
// src/utils/format.test.ts
import { describe, it, expect } from 'vitest'
import { formatBytes } from './format'

describe('formatBytes', () => {
  it('formats bytes correctly', () => {
    expect(formatBytes(1024)).toBe('1.0 KB')
    expect(formatBytes(1024 * 1024)).toBe('1.0 MB')
  })
})
```

### Run Tests

```bash
npm run test
npm run test:watch
npm run test:coverage
```

## Debugging

### Backend Debugging

Enable debug logs:

```bash
DEBUG=mixcode:* npm run server
```

Or use node inspector:

```bash
node --inspect ./server/src/index.ts

# Then open chrome://inspect in Chrome
```

### TUI Debugging

Run with React dev tools:

```bash
REACT_DEBUG=true npm run tui
```

### WebSocket Debugging

Monitor network traffic:

```bash
# In browser (if running in browser mode)
# Open DevTools → Network → WS
```

Or use wscat:

```bash
npm install -g wscat
wscat -c ws://127.0.0.1:4567/ws
```

## Building for Production

### Build Outputs

```bash
# Build server
npm --workspace server run build

# Build TUI
bun --cwd tui run build

# Both
npm run build
```

Outputs:
- Server: `/server/dist`
- TUI: `/tui/dist`

```bash
docker build -t mixcode:latest .
docker run -p 4567:4567 mixcode
```

## Troubleshooting Development

### "Module not found" errors

```bash
# Reinstall dependencies
rm -rf node_modules package-lock.json bun.lock
npm install
```

### "Port already in use"

```bash
# Use different port
PORT=4568 npm start
```

### "TypeScript errors"

```bash
# Check types across workspace
npm run typecheck

# Fix individual workspace
cd server && npx tsc --noEmit
```

### "Build failures"

```bash
# Clean and rebuild
rm -rf server/dist tui/dist
npm run build
```
