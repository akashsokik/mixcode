# Collectives

`server/src/collectives/` is a library for fan-out workloads built on the
Anthropic SDK. It lets one orchestrator agent dispatch work to hundreds of
short-lived "workers" through MPI-flavored primitives — `scatter_map`,
`map_reduce`, `all_reduce`, `tree_reduce` — without spawning subprocesses or
full agent loops per task.

## Why not just delegate to subagents?

The existing `delegate_run` / `task_spawn` tools (in `server/src/runners/delegate.ts`)
spawn a full peer agent: own session, own working directory, own SDK
control-protocol loop. That overhead is fine for 1–10 delegations but
prohibitive at 100s.

Collectives takes the opposite approach. Each worker is a single
`messages.create` call on the raw Anthropic SDK — no tool use, no
multi-turn, no filesystem. You can fan out 100s of them in one Node event
loop, with one global concurrency semaphore.

## When to use which

| Need                                      | Reach for           |
| ----------------------------------------- | ------------------- |
| Peer agent that uses tools / files        | `delegate_run`      |
| Structured fan-out with peer agents       | `task_spawn`        |
| 10s–100s of single-shot completions       | `scatter_map`       |
| Consolidate many completions into one     | `map_reduce`        |
| Multi-agent consensus / debate            | `all_reduce`        |
| Fan-in too big for one reducer's context  | `tree_reduce`       |
| Latency-tolerant batch (24h)              | Anthropic Batches   |

## Architecture

```
┌──────────────────────────────────────────────┐
│   Orchestrator agent (Opus, full tools)      │
│   - Plans + dispatches                       │
└─────────────────────┬────────────────────────┘
                      │ MCP tool call
              ┌───────▼────────┐
              │ collectives MCP│
              │   (in-process) │
              └───────┬────────┘
                      │
              ┌───────▼────────┐
              │  WorkerPool    │  concurrency cap
              │  (semaphore)   │  + per-task isolation
              └───────┬────────┘
                      │ async fan-out
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
   ┌────────┐   ┌────────┐    ┌────────┐
   │ worker │   │ worker │... │ worker │   raw messages.create
   │ (Haiku)│   │ (Haiku)│    │ (Haiku)│   shared system + cache
   └────────┘   └────────┘    └────────┘
```

The orchestrator sees one tool call → one structured JSON result. It does
not see 200 streaming partials, which would blow its context window.

## Primitives

### `scatter_map`

Apply one task template to N inputs in parallel.

```ts
import { AnthropicWorker, scatterMap } from "./collectives/index.js";

const worker = new AnthropicWorker();
const { texts, stats } = await scatterMap(worker, {
  system: "You are a terse summarizer. One sentence per item.",
  template: "Summarize: {input}",
  inputs: docs, // array of strings
});
```

Shared system prompts are prompt-cached automatically — at scale this is
what makes fan-out economical (cache hits cost ~10% of input tokens and
don't count against ITPM).

### `map_reduce`

Fan out, then consolidate to one output via a reducer prompt. The reducer
typically wants a smarter model than the mappers.

```ts
const { text } = await mapReduce(worker, {
  mapTemplate: "Extract themes from: {input}",
  inputs: docs,
  reduceTemplate: "Identify the 3 dominant themes:\n{items}",
  reduceModel: "claude-sonnet-4-6",
});
```

### `all_reduce`

Two-round consensus. Each worker drafts an answer, then every worker is
shown all drafts and asked to refine. Useful for multi-agent debate or
ensemble reasoning.

```ts
const { refined, drafts } = await allReduce(worker, {
  prompts: questions, // one prompt per worker
  refineTemplate:
    "Your draft:\n{ownDraft}\n\nAll drafts:\n{allDrafts}\n\nProduce a refined answer.",
});
```

### `tree_reduce`

Hierarchical reduction. Maps in parallel, then collapses in waves of
`branchFactor` until one result remains. Use when fan-in is too big to fit
one reducer call's context.

```ts
const { text, levels } = await treeReduce(worker, {
  mapTemplate: "Summarize: {input}",
  inputs: documents, // hundreds of items
  reduceTemplate: "Merge:\n{items}",
  branchFactor: 8,
});
```

## Wiring into an orchestrator agent

The MCP server factory in `collectives/mcp.ts` exposes the primitives as
tools the Agent SDK can call. Register it via the `mcpServers` option:

```ts
import { query } from "@anthropic-ai/claude-agent-sdk";
import { AnthropicWorker, createCollectivesMcpServer } from "./collectives/index.js";

const worker = new AnthropicWorker();
const collectives = createCollectivesMcpServer({ worker, concurrency: 15 });

for await (const msg of query({
  prompt: "Classify each of these 200 PR titles by area.",
  options: {
    model: "claude-opus-4-7",
    mcpServers: { collectives },
    allowedTools: [
      "mcp__collectives__scatter_map",
      "mcp__collectives__map_reduce",
      "mcp__collectives__all_reduce",
      "mcp__collectives__tree_reduce",
    ],
  },
})) {
  // ...
}
```

`server/src/example-collectives.ts` runs this end-to-end against the real
API. `server/src/smoke-collectives.ts` exercises the library directly
without going through an orchestrator agent.

## Scaling limits

| Limit                              | Mitigation                                  |
| ---------------------------------- | ------------------------------------------- |
| 15 concurrent reqs per API key     | Rotate keys; or use Batches API for non-realtime |
| 429 / overloaded_error             | Built-in exponential backoff with jitter    |
| Token budget back to orchestrator  | Use `map_reduce` / `tree_reduce` to collapse |
| Mapper context too big             | Truncate inputs; or chunk into rounds       |

## Tests

```
npm --workspace server run test
```

Unit tests use a deterministic `MockWorker` — no API key required. The
smoke and example scripts (`smoke:collectives`, `example:collectives`) hit
the real API and need `ANTHROPIC_API_KEY` set.
